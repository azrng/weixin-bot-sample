# 项目设计文档

## 1. 微信官方 iLink Bot API 演示 Demo

### 1.1 架构设计

- 技术栈采用 `.NET 10 + Blazor Server + BootstrapBlazor`，以单项目方式同时承载 UI、服务层与本地持久化，优先保证 Demo 可快速落地和稳定演示。
- 项目采用轻量分层：
  - `Components/Pages` 负责演示页面与交互
  - `Services` 负责微信 API 调用、消息监听、主动推送和固定回复编排
  - `Models` 负责配置、绑定会话、消息记录、日志 DTO
  - `Storage` 负责 JSON 文件持久化
- 微信协议处理优先复用 `C:\Work\github\OpenCowork` 中官方微信渠道服务的成熟实现思路，并裁剪为本 Demo 的核心能力：
  - 获取二维码 `get_bot_qrcode`
  - 轮询扫码结果
  - 长轮询获取消息 `getupdates`
  - 发送文本消息 `sendmessage`
- 本 Demo 不实现登录页和权限系统，默认直接进入演示页，减少与演示目标无关的流程。
- 为保证刷新后状态仍可见，一期采用 `App_Data` 下的 JSON 文件保存：
  - 基础配置
  - 绑定结果
  - 最近入站消息
  - 最近回复记录
  - 最近主动推送记录
  - 操作日志

### 1.2 功能需求

#### 1.2.1 模块划分

- 演示框架模块
  - 应用图标与应用名称
  - 页面头部说明
- 微信 Bot 演示模块
  - 基础配置
  - 微信绑定
  - 监听控制
  - 主动推送
  - 入站消息记录
  - 固定祝福语回复
  - 日志展示

#### 1.2.2 业务规则

- 页面默认直接进入 Demo 主页面。
- 微信绑定成功后，系统立即持久化 `Token`、`AccountId`、`UserId`、绑定状态、绑定时间。
- 启动监听前必须校验基础配置是否完整。
- 系统同一时刻只允许一个监听实例运行。
- 入站消息只处理文本消息；非文本消息记录为“已忽略”。
- 收到文本消息后，默认回固定祝福语。
- 主动推送属于显式按钮操作，执行后必须写入结果记录。
- 页面上的“绑定微信”“刷新二维码”“启动监听”“停止监听”“推送演示消息”必须是真实有效的操作。

#### 1.2.3 固定回复规则

- 一期不接入 AI。
- 默认回复模板为固定祝福语，例如：
  - `祝您今天顺顺利利，万事如意。`
  - `感谢您的消息，祝您生活愉快、事事顺心。`
- 页面可展示当前固定回复模板，便于演示说明。

### 1.4 交互说明

#### 1.4.1 配置保存交互

- 演示操作员修改基础配置后点击“保存配置”立即落盘。
- 保存成功后更新页面摘要状态。
- 若关键字段缺失，则提示错误并阻止启动监听。

#### 1.4.2 微信绑定交互

- 点击“绑定微信”后调用二维码接口。
- 若已有未过期绑定会话，则优先复用；点击“刷新二维码”强制重新获取。
- 页面展示二维码图片、扫码说明、会话 Key 与开始时间。
- 后台轮询扫码结果，成功后自动更新绑定状态与基础配置字段。
- 若二维码过期，则界面提示“二维码已过期”，并保留刷新入口。

#### 1.4.3 监听交互

- 点击“启动监听”时，系统先校验绑定状态和关键配置。
- 启动成功后运行后台长轮询服务并写入运行时间。
- 点击“停止监听”时取消轮询并写入停止时间。
- 若轮询异常但可恢复，则自动重试并写日志；若不可恢复，则状态切换为错误。

#### 1.4.4 用户发消息自动回复交互

- 当长轮询收到文本消息时，系统生成一条入站消息记录。
- 系统根据固定模板生成祝福语回复。
- 系统调用 `sendmessage` 回发给微信用户。
- 页面自动刷新最近消息和最近回复区域，保证演示可见。
- 若消息缺少 `context_token`，则仅记录消息和失败原因，不执行回包。

#### 1.4.5 主动推送交互

- 操作员输入 `ExternalChatId`、`ContextToken` 和消息内容。
- 点击“推送演示消息”后调用 `sendmessage`。
- 页面显示发送结果、响应摘要和发送时间。
- 结果写入主动推送记录和日志区，便于演示。

#### 1.4.6 边界与错误处理

- API 请求失败：展示错误提示、保留最近一次成功状态、写日志。
- JSON 文件损坏：页面进入错误态，并提供重置示例数据入口。
- 二维码轮询超时：展示失败信息并保留重新生成入口。
- 回包失败：保留入站消息，并在回复结果列中标记失败原因。

### 1.5 阶段边界

- 本设计文档对应阶段 0。
- 用户确认后进入阶段 1，实现 Blazor Server 项目骨架、单页 Demo、服务层、本地持久化与真实微信 API 示例链路。
