# 微信官方 iLink Bot API 示例需求文档

## 1. 项目背景

当前仓库尚未开始实际开发。用户希望基于微信官方 iLink Bot API 编写一个可运行的示例项目，用于演示官方微信机器人接入链路，并要求界面上能直观看到绑定微信、测试推送消息、用户发送消息以及系统回复消息的完整流程。

用户同时指定可参考 `C:\Work\github\OpenCowork` 中现有的微信推送渠道代码，说明目标不是静态原型，而是复用既有协议接入经验，做一个更聚焦、更容易演示和验证的独立示例。

## 2. 目标与范围

### 2.1 目标

- 提供一个基于 Blazor Server 的微信官方 iLink Bot API 使用示例。
- 示例项目必须可以展示并操作真实的微信 Bot 接入主链路。
- 用户可以在页面内完成微信绑定、测试消息推送、查看入站消息、触发自动回复。
- 示例中的关键状态在浏览器刷新后依然可见，不能只是前端假交互。

### 2.2 一期范围

- 登录页与应用主框架
- 左侧多级菜单与管理员默认全权限
- 微信渠道示例页
- iLink Bot 基础配置管理
- 二维码绑定微信
- 绑定状态展示
- 手动测试推送消息
- 长轮询接收微信用户文本消息
- 自动回复开关与文本回包
- 消息日志与最近交互记录展示
- 基于本地文件的真实持久化

### 2.3 非本期范围

- 多渠道接入
- 图片、语音、文件等多媒体消息处理
- 复杂权限系统
- 企业级多租户
- 外部数据库与 Docker 部署编排
- AI 模型编排与复杂工具调用

## 3. 用户角色

### 3.1 管理员

- 登录系统
- 配置 iLink Bot 连接参数
- 发起微信绑定
- 启动或停止消息监听
- 发送测试消息
- 查看入站消息与回复日志
- 开关自动回复
- 清空或刷新测试数据

### 3.2 微信用户

- 通过微信扫描二维码完成绑定
- 向机器人发送文本消息
- 接收系统自动回复

## 4. 页面与模块清单

### 4.1 登录页

- 显示应用图标、应用名称、系统简介
- 提供默认管理员账号登录入口
- 登录成功后进入系统首页

### 4.2 首页 / 仪表盘

- 展示示例项目说明
- 展示微信渠道当前运行状态
- 展示最近一次绑定时间、最近一次消息时间
- 提供进入“微信 Bot 示例”页面的快捷入口

### 4.3 微信 Bot 示例页

- 基础配置区
  - 网关地址
  - RouteTag
  - Token
  - 账号 ID
  - 用户 ID
- 微信绑定区
  - 生成二维码
  - 刷新二维码
  - 轮询等待绑定
  - 展示绑定账号、绑定状态、绑定时间
- 运行控制区
  - 启动监听
  - 停止监听
  - 展示运行状态和错误信息
- 测试推送区
  - 输入会话 ID / `context_token`
  - 输入测试消息
  - 发送测试消息
  - 展示发送结果
- 入站消息区
  - 展示最近收到的微信文本消息
  - 展示消息发送者、会话、时间、原文
- 自动回复区
  - 自动回复开关
  - 默认回复模板
  - 最近一次回复内容
- 交互日志区
  - 展示二维码获取、绑定成功、监听状态、收消息、回消息等关键日志

## 5. 核心业务流程

### 5.1 登录流程

1. 管理员打开登录页。
2. 输入默认账号信息并登录。
3. 系统写入当前登录状态并跳转首页。

### 5.2 微信绑定流程

1. 管理员进入微信 Bot 示例页。
2. 填写或确认 iLink Bot 基础配置。
3. 点击“绑定微信”或“生成二维码”。
4. 系统调用 iLink API 获取二维码并展示。
5. 管理员使用微信扫码。
6. 系统轮询扫码状态，成功后保存 Token、账号 ID、用户 ID、绑定账号名与绑定时间。
7. 页面刷新后仍可看到已绑定状态。

### 5.3 测试推送流程

1. 管理员在测试推送区输入目标会话与消息内容。
2. 点击“发送测试消息”。
3. 系统调用 iLink `sendmessage` 接口。
4. 页面展示发送结果、HTTP 状态、响应摘要与发送时间。
5. 发送记录写入本地持久化日志。

### 5.4 用户发消息并自动回复流程

1. 管理员启动监听。
2. 系统长轮询 `getupdates` 获取用户消息。
3. 当收到文本消息时，系统保存入站消息到本地存储。
4. 若自动回复已开启，则根据默认回复模板生成回复文本。
5. 系统调用 `sendmessage` 将回复回发给微信用户。
6. 页面实时或刷新后可查看该次入站消息和回复结果。

## 7. 权限与可见范围

- 管理员默认拥有所有菜单权限。
- 一期仅实现管理员角色。
- 未登录用户只能访问登录页。
- 已登录管理员可访问首页和微信 Bot 示例页全部功能。

## 8. 边界场景与异常

- 未配置基础参数时禁止启动监听，并给出明确提示。
- 二维码过期时允许刷新重新生成。
- 绑定轮询超时时给出失败说明，并保留重试入口。
- `getupdates` 请求超时时，系统自动重试并记录日志。
- 消息没有 `context_token` 时，不执行回包并记录原因。
- `sendmessage` 失败时，页面展示错误详情且保留该条消息记录。
- 页面加载不到配置或日志文件时，展示错误态并允许重试。

## 9. 非功能要求

- 项目使用 UTF-8 编码。
- 所有页面必须有 `loading`、`empty`、`error`、`no-permission` 状态。
- 所有按钮必须为真实可交互动作。
- 刷新浏览器后仍能看到配置、绑定状态、最近消息与日志。
- 优先复用 `OpenCowork` 中已验证的微信协议处理逻辑，避免重新猜测协议。
