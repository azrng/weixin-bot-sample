# 微信官方 iLink Bot API 演示 Demo 需求文档

## 1. 项目背景

当前仓库用于编写一个微信官方 iLink Bot API 的演示 Demo。这个 Demo 的目标不是做完整后台系统，而是做一个可以现场演示的最小可用示例，让观众看到：

- 如何绑定微信 Bot
- 用户给 Bot 发消息后，系统如何自动回复
- 系统如何主动给用户推送一条消息

用户指定可以参考 `C:\Work\github\OpenCowork` 中现有的微信推送渠道代码，因此本项目优先复用已经验证过的协议处理经验，而不是重新摸索接口细节。

## 2. 目标与范围

### 2.1 目标

- 提供一个基于 Blazor Server 的微信官方 iLink Bot API 演示 Demo。
- Demo 页面直接展示微信绑定、主动推送、入站消息与自动回复结果。
- 用户在微信里给 Bot 发消息后，Bot 能返回固定祝福语。
- 页面刷新后仍能看到最近配置、绑定状态、消息记录和推送记录。

### 2.2 一期范围

- 单页演示界面或极简导航界面
- 应用图标与应用名称
- iLink Bot 基础配置管理
- 二维码绑定微信
- 绑定状态展示
- 启动监听与停止监听
- 长轮询接收微信文本消息
- 固定祝福语自动回复
- 主动推送消息示例按钮
- 最近消息记录与操作日志展示
- 基于本地文件的真实持久化

### 2.3 非本期范围

- 登录页
- 权限系统
- 多用户管理
- 多渠道接入
- 图片、语音、文件等多媒体消息处理
- AI 大模型回复
- 外部数据库与复杂部署方案

## 3. 用户角色

### 3.1 演示操作员

- 配置 iLink Bot 基础参数
- 发起微信绑定
- 启动或停止监听
- 点击按钮主动推送一条演示消息
- 查看用户发来的消息
- 查看 Bot 自动回复结果

### 3.2 微信用户

- 扫码完成 Bot 绑定
- 给 Bot 发送文本消息
- 接收 Bot 返回的固定祝福语

## 5. 核心业务流程

### 5.1 微信绑定流程

1. 演示操作员打开 Demo 页面。
2. 填写或确认 iLink Bot 基础配置。
3. 点击“绑定微信”或“生成二维码”。
4. 系统调用 iLink API 获取二维码并展示。
5. 用户使用微信扫码。
6. 系统轮询扫码状态。
7. 绑定成功后保存 Token、账号 ID、用户 ID、绑定账号名和绑定时间。
8. 页面刷新后仍显示已绑定状态。

### 5.2 用户发消息自动回复流程

1. 演示操作员点击“启动监听”。
2. 系统长轮询 `getupdates` 获取微信用户消息。
3. 当收到文本消息时，系统将消息保存到本地记录。
4. 系统按固定模板生成祝福语，例如“祝您今天顺顺利利，万事如意。”
5. 系统调用 `sendmessage` 将祝福语回发给微信用户。
6. 页面展示最近收到的消息和最近一次回复结果。

### 5.3 主动推送流程

1. 演示操作员在主动推送区填写目标 `ExternalChatId`、`ContextToken` 和消息内容。
2. 点击“推送演示消息”。
3. 系统调用 iLink `sendmessage` 接口向指定用户或会话发送消息。
4. 页面展示发送结果、响应摘要和发送时间。
5. 推送记录写入本地日志并在页面可见。

## 7. 可见范围

- Demo 对外只提供一个演示页面。
- 不实现登录和权限控制。
- 所有操作入口都集中在同一页面，方便演示。

## 8. 边界场景与异常

- 未配置基础参数时禁止启动监听，并给出明确提示。
- 二维码过期时允许刷新重新生成。
- 绑定轮询超时时给出失败说明，并保留重试入口。
- `getupdates` 请求超时时，系统自动重试并记录日志。
- 消息没有 `context_token` 时，不执行回包并记录原因。
- `sendmessage` 失败时，页面展示错误详情且保留该条操作记录。
- 页面加载不到配置或日志文件时，展示错误态并允许重试。

## 9. 非功能要求

- 项目使用 UTF-8 编码。
- 页面必须有 `loading`、`empty`、`error` 状态。
- 所有按钮必须为真实可交互动作。
- 刷新浏览器后仍能看到配置、绑定状态、最近消息与日志。
- 优先复用 `OpenCowork` 中已验证的微信协议处理逻辑，避免重新猜测协议。

## 10. 已确认约束

- 不需要登录页。
- 自动回复暂时固定为祝福语，不接 AI。
- Demo 的重点是“微信绑定成功后可以正常收发消息”和“演示主动推送”。
